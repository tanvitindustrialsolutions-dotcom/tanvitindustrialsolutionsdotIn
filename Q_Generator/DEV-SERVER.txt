WHY: Opening index.html as file:/// causes browser security errors and breaks auth/storage.

HOW TO RUN (pick one):

1) Double-click: start-local.bat
   Then open: http://localhost:5500

2) In this folder, run:
   npm install
   npm start
   Open: http://localhost:5500

3) If you have Python only:
   python -m http.server 5500
   Open: http://localhost:5500

SUPABASE: No dashboard changes are required just to run the app on localhost.
Your existing project URL + anon key work for normal sign-in and data.

Only if you use email links (password reset / signup confirm) that must open
localhost would you add http://localhost:5500 under Redirect URLs — optional.

EDGE "Tracking prevention blocked storage": often from third-party CDNs (fonts, etc.).
Use http://localhost (not file://). If it persists, Edge Settings - Cookies - add
localhost exception or use Chrome for testing.

"Host validation failed" / "Host is not supported" / "not in insights whitelist"
(console shows content.js or read.js — NOT app.js):
Those messages are from a BROWSER EXTENSION. Many extensions refuse to run on
http://localhost:5500 and may interfere with the page or network.
Fix: Open the site in InPrivate/Incognito with extensions disabled, OR in your
browser's extension settings turn OFF or "don't run on localhost" for tools like
Grammarly, Microsoft Editor, Read/Reader extensions, shopping helpers, etc.
Our app does not perform host validation.
